Platform Engineer Workflow

  • In this chapter, as a Platform Engineer, you’ll configure DNS and TLS for the Gateway API. You will implement essential policies for managing API traffic like:

    • DNS Policy

    • TLS Policy

    • Authentication policy

    • Rate Limiting Policy

Objectives

In this chapter, you will:

  • Set up DNS and TLS for Gateway API

  • Create Auth Policy

  • Create TLS Policy

  • Create Rate Limit Policy

  • Create DNS Policy

  • Create HTTP Route

Before starting this lab, ensure you have completed the previous chapter on installing and deploying Connectivity Link in Red Hat OpenShift cluster. This step is critical as it sets the foundation for the upcoming exercises by providing you with a properly configured environment.
