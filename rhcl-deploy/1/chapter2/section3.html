Install and configure Istio with the OpenShift Service Mesh

Kuadrant integrates with Istio as a Gateway API provider.

In this lab, you install and set up an Istio-based Gateway API provider using the OpenShift Service Mesh Operator.

  • Install Red Hat Openshift Service Mesh 3

    1. In your Red Hat Openshift console , navigate to Operators option. Select OperatorHub option and type "service mesh" in the search box. Select Red Hat OpenShift Service Mesh 3 that appears in the search result, as shown in the image below:

    2. Confirm all the options are same as image below and select Install button:

    3. Note that you can now see Red Hat OpenShift Service Mesh 3 operator in Red Hat Openshift console’s Installed Operators section.

  • Configure Istio Gateway API provider.

    1. To configure the Istio Gateway API provider, apply the yaml settings using below command:

      oc apply -f -<<EOF
apiVersion: sailoperator.io/v1alpha1
kind: Istio
metadata:
  name: default
spec:
  namespace: gateway-system
  updateStrategy:
    type: InPlace
    inactiveRevisionDeletionGracePeriodSeconds: 30
  version: v1.23.0
  values:
    pilot:
      autoscaleEnabled: false
EOF

      You should see the output as follows:

      istio.sailoperator.io/default created

    2. Wait for Istio to be ready and confirm the default istio provider is available.

      oc get istio -n gateway-system

      You should see the output as follows:

      NAME      REVISIONS   READY   IN USE   ACTIVE REVISION   VERSION   AGE
default   1           1       0        Healthy           v1.23.0   2m28s

      Once you notice the ACTIVE REVISION is Healthy output for above command, proceed with next section.
